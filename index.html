Sparkloop

The end-to-end flow for turning audience signals, artefacts, and performance data into production-ready content angles + scripts.

WorldFirst social v1 · 2026-05-13 Reference run · SEA brand channel
01

01What Sparkloop is

Sparkloop is the upstream content engine. It takes signals from the world (or our own data), runs them through human + AI iteration, and outputs ready-to-script content angles plus production briefs.

Three stages, in order:

  • Initiate — kick off from any of 8 input sources
  • Iterate — run the brainstorm board (v1 → v(n) → wrap with banked angles list)
  • Refine + ship — deep research per topic batch, then 5 script variants per piece. The production handoff happens only when scripts are confirmed.

End of Sparkloop = scripts confirmed. Everything after that is production.

Reference run: SEA brand channel · 4 iterations · 30 banked pieces · research + scripts pending.

02

02The flow at a glance

Four entry skills feed into /spark_idea. Three are manual; one (/spark_scan) is an automated radar that fans out across five signal sources. Iteration runs until banked outweighs iterating. Wrap hands off to /spark_listen for topical research, then to /spark_script for production-ready scripts.

Note: most initiators are existing skills. Three are new wrappers or fresh builds. See section 08 for build status.

03Initiators

Four entry points kick off a Sparkloop. Three are manual (you bring the input). The fourth (/spark_scan) is an automated radar that fans out across five signal sources and surfaces them in one dashboard.

All four feed the same downstream pipeline: /spark_idea → /spark_listen → /spark_script.

04

04Spark_idea — the brainstorm loop

The iteration heart. Mobile-first HTML board generated per topic. v1 produces 20-30 angle candidates organised by pillar. The user ticks formats, comments per card, and flags "find more angles like this." Auto-categorisation sorts cards into three states.

Three states, auto-assigned

  • Banked — format ticked, no further notes. Locked in for production.
  • Iterating — has a comment or a "find more" flag, or no format ticked yet but engaged. Still shaping.
  • Archived — no engagement at all. Set aside.

The loop

Generated prompt at the bottom of the page auto-builds based on selections. Copy, paste back to the chat, AI generates v(n+1): rewrites cards per your notes, generates 5 new derivatives per "find more" flag, drops stale signals, banks finalised picks. Repeat until banked outweighs iterating, then wrap.

What wrap produces

Wrap of /spark_idea produces a banked angles brief — the list of locked-in pieces with their format, talent, bilingual flag, and shortlist reasoning. This is an interim deliverable. It feeds /spark_listen and /spark_script. It is not the production handoff.

Reference run

SEA brand channel. 4 iterations (v1 → v4). Started with 20 angles. Ended with 30 banked pieces (2 long-form anchors, 27 short-form reels, 1 KOL angle). Banked-angles brief shipped. Research and scripts pending; production handoff happens only after scripts confirm.

05

05Spark_listen — deep research

After wrap, before scripts. Bridge skill between angle pick and script writing. Built on /wf_focused_listen.

Batching strategy

Running deep research 30 times for 30 angles is wasteful. The angles cluster naturally into 6-8 research themes. One /spark_listen run per theme produces one structured brief that feeds 3-9 angles.

Example (SEA batch): 30 angles clustered into 7 themes — CNY rails, FX margin, wire rejection, paying-in-CNY economics, personal-vs-business tools, WF product mechanics, switch-point math. Seven research runs, not thirty.

Output

One brief per theme. Statistics, case studies, authority quotes, real-world examples. Feeds /spark_script with the evidence-base it needs to write scripts that ground in facts rather than fabricated framing.

06

06Spark_script — script generation

Future stage. For each banked piece, produces five distinct script variants. The page shows variant 1 by default; a dropdown reveals variants 2-5 for alternatives.

Variants vary by format

  • Long-form — 5 structural directions (sequential, problem-led, day-in-life, cost-decomposition, question-led)
  • Short-form — 5 hook + body combinations using different psychological levers
  • KOL angle — 5 distribution framings (standalone math, co-branded carousel, native deep dive, customer conversation, LinkedIn longform)

Hands off to /wf_yt_script, /new_reel_scripts_x5_v1, and /wf_quick_reel for final voiceover-ready scripts.

End of Sparkloop

When scripts are confirmed, /spark_script produces the production handoff doc. That is the terminal deliverable of Sparkloop. Talent booking, studio scheduling, calendar slotting, and posting all happen downstream and outside Sparkloop.

07

07What Sparkloop isn't

Sparkloop is the angle-discovery and selection layer. Everything downstream is a separate skill.

  • Doesn't write final voiceover scripts — handoff to /wf_yt_script, /new_reel_scripts_x5_v1, /wf_quick_reel
  • Doesn't book talent — manual outreach drafted via /email / /reply
  • Doesn't book studio time
  • Doesn't manage the publishing calendar — /cal_add / /cal_update
  • Doesn't push to social channels — /posting
  • Doesn't run paid amplification — /boosted or the Meta Ads MCP

The output of Sparkloop is locked-in angles and script variants. Production owns the rest.
